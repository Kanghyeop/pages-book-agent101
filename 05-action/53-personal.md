지금까지 AI로 돈 버는 10가지 방법을 살펴봤습니다. 그런데 여기서 멈추면 "좋은 이야기 들었다"로 끝납니다.

이제부터는 구체적인 액션플랜입니다. 당장 내일부터 뭘 해야 하는지. 이 섹션은 완벽하게 수행해야 하는 투두리스트가 아닙니다. 방향을 아직 잡지 못했을 때 참고할 수 있는 경험 목록이자 체크리스트에 가깝습니다.

## 1. 지금 당장 할 수 있는 6가지

### 첫째, 퍼스널 브랜딩을 시작하라

직장인이라도 조직 안의 역할에만 자신을 가두지 마세요. 남들과 구별되는 강점과 매력을 언어로 정리해야 합니다. 이건 이직이나 창업을 위한 포장이 아닙니다. 변화가 잦은 시대에 내 가치를 안정적으로 전달하는 생존 기술입니다.

> "나는 마케팅팀 대리입니다" — 이건 직함입니다.
>
> "저는 B2B SaaS의 콘텐츠 마케팅을 하고, 특히 기술 제품을 비전공자에게 설명하는 걸 잘합니다" — 이게 브랜딩입니다.
>
> 왜 필요한가: 이직할 때 설명이 쉬워집니다. 협업 요청이 들어옵니다. 변화가 잦은 시대에 내 가치를 안정적으로 전달할 수 있습니다.

---

### 둘째, AI 어시스턴트 하나는 유료로 쓰라

ChatGPT든, Claude든, Gemini든. 무료 체험은 '가능성'만 보고 끝나기 쉽습니다. "오 신기하네, 이런 것도 되네" 하고 몇 번 써보다가 잊어버립니다.

유료 구독은 다릅니다. 돈을 내면 심리가 바뀝니다. "이왕 돈 냈으니 뽕을 뽑아야지"라는 생각이 반복 사용을 만들고, 반복 사용이 습관을 만들고, 습관이 실력을 만듭니다. 어느 순간 AI 어시스턴트 없이 일하는 게 어색해집니다. 그때부터 진짜 생산성 변화가 시작됩니다.

일단 한 달만 결제해보세요. 그리고 한 달을 아깝지 않게 써보세요. 정말 더 똑똒해진 본인을 만나게 될 것입니다.

> 무료 체험의 한계: "오 신기하네" → 끝. 사용량 제한에 걸림. 진짜 업무에 못 써봄. 가능성만 보고 종료.
>
> 유료 구독의 효과: "돈 냈으니 써야지" → 반복 사용. 반복하니까 요령 생김. 요령 생기니까 실력 됨. 실력 되니까 성과 남.
>
> 월 2만원이 아깝다고요? 그 2만원으로 월 10시간 절약하면 시급 2천원. 최저시급보다 싸게 일 시키는 겁니다.

그리고 무료 사용자에게는 유료 사용자에 비해 더 안 좋은 모델을 제공할 확률이 높습니다. '현 시점 가장 좋은 모델은 이런게 가능하구나'라는 것을 이해할 기회를 잠재적으로 박탈당합니다.

---

### 셋째, 학습 예산을 실제로 집행하라

중요한 건 지출 자체가 목적이 아니라는 점입니다. 돈을 쓰면서 뭔가를 만들어야 합니다. 콘텐츠를 제작하든, 자동화를 구축하든, 실험적 프로젝트를 돌려보든. "비용 → 교훈 → 성과"의 루프를 만들어야 합니다.

예를 들어 API 비용 5만 원을 써서 뉴스레터 자동 요약 시스템을 만들었다고 해봅시다. 작동이 잘 되면 성과고, 실패하면 교훈입니다. 둘 다 남는 게 있습니다. 하지만 "언젠가 써봐야지" 하면서 구독만 해두고 안 쓰면, 돈만 나가고 아무것도 안 남습니다.

> 나쁜 예: 강의 질렀는데 안 봄. API 크레딧 샀는데 안 씀. 책 샀는데 안 읽음. → 지출만 있고 학습 없음.
>
> 좋은 예: 10만원 API로 자동화 실험 → 뭐가 되고 안 되는지 앎. 50만원 들여 콘텐츠 제작 → 조회수/반응 데이터 얻음.
>
> 핵심 루프: 비용 → 실험 → 교훈 → 성과 → 재투자

부가가치를 만드는 사람이 되어야 합니다. AI에 돈을 쓰는 사람은 많지만, 그 돈으로 뭔가를 만들어내는 사람은 드뭅니다.

---

### 넷째, 새 모델이 나오면 직접 써봐라

GPT-4가 나왔을 때, Claude 3가 나왔을 때, Gemini가 업데이트됐을 때. 최신 모델이 출시되면 반드시 직접 써보세요. 뉴스 기사나 벤치마크 점수만 보지 말고, 내 업무에 실제로 적용해보세요.

"이전 버전에서 안 되던 게 이제 되네", "이건 오히려 퇴보했네", "이 작업은 이 모델이 더 낫네". 이런 감각은 직접 써봐야만 생깁니다.

> "GPT-5 나왔대" → "아 그래?" → 끝. 이러면 영원히 감 못 잡습니다.
>
> 해야 할 것: 나오자마자 직접 써봅니다. 같은 프롬프트로 이전 모델과 비교합니다. 뭐가 달라졌는지 기록합니다.
>
> 이 체감이 쌓이면: 같은 비용으로 더 좋은 결과. 작업별 최적 모델 선택 가능. 기술 변화가 업무에 미치는 영향 예측 가능.

모델 간 차이를 이해하면 같은 비용으로 더 높은 품질을 얻을 수 있습니다. 간단한 작업에 비싼 모델을 쓸 필요 없고, 복잡한 작업에 저렴한 모델을 쓰면 결과물이 망가집니다. 어떤 작업에 어떤 모델이 맞는지 아는 것, 이게 실력입니다.

---

### 다섯째, 프롬프트를 스킬로 바꿔라

한 번 잘 작동한 프롬프트를 그냥 흘려보내지 마세요. 보관하세요. 템플릿, 체크리스트, 자동화 스크립트로 바꿔서 재사용 가능하게 만드세요. 한 번의 성과를 반복 가능한 역량으로 전환하는 사람이 되어야 합니다.

단, 한두번 하고 넘어갈 1회성 업무가 아닌 매일 반복해서 하는 업무를 이야기합니다.

> 1회성: "이 PDF 요약해줘" → 결과 받음 → 끝
>
> 재사용: "PDF 요약 프롬프트 + 출력 형식 + 검증 기준" → 다음에 바로 적용 가능 → 다른 사람도 쓸 수 있음
>
> "한 번의 성과"를 "반복 가능한 역량"으로 전환하십시오.

---

### 여섯째, 작은 것 하나라도 완전 자동화해서 운영하라

간단한 작업 하나, 혹은 소규모 업무 흐름 하나를 완전히 자동화해서 실제로 돌려보세요. 자동화는 만들어두고 끝내는 게 아닙니다. 장애, 예외, 품질 문제를 다루며 계속 개선하는 '운영 경험'이 쌓일 때 진짜 실력이 됩니다.

> "만들기"와 "운영"은 완전히 다른 기술입니다.
>
> 만들기: 한 번 성공하면 끝. 예쁜 데모 가능. "와 됐다!" 순간의 쾌감.
>
> 운영: 매일 돌아가야 함. 장애 나면 고쳐야 함. 예외 케이스 처리해야 함. 비용 모니터링해야 함. 품질 떨어지면 개선해야 함.
>
> 시작 추천: 규모는 나 혼자 또는 2-3명 팀. 범위는 주 1회 정도 돌아가는 단순한 것. 기간은 최소 3개월 운영.
>
> "자동화 만들 줄 알아요" vs "자동화 3개월 운영해봤어요" — 후자가 10배 가치 있습니다.

## 2. 세 가지 역할 중 하나를 선택하라

AI와 함께 일하는 환경에서, 당신은 다음 셋 중 적어도 하나의 역할 정체성을 가져야 합니다.

| 역할 | 설명 | 핵심 역량 |
| --- | --- | --- |
| M자형 감독자 | 여러 도메인을 폭넓게 이해하는 제너럴리스트. 에이전트와 사람을 함께 배치해 하이브리드 팀을 조율한다. | 전체 그림 파악, 자원 배분, 커뮤니케이션 |
| T자형 전문가 | 깊은 전문성을 바탕으로 워크플로를 설계하고 예외를 처리한다. 결과물의 품질과 책임을 진다. | 도메인 지식, 판단력, 품질 관리 |
| AI 강화 실행자 | 영업, 서비스, 운영 등 현장에서 에이전트를 활용해 성과를 만들어내는 실행자. 시스템 작업을 줄이고 사람과의 상호작용에 집중한다. | 실행력, 도구 활용, 현장 감각 |

핵심은 단순히 AI를 "쓰는 사람"이 아니라, AI와 함께 일하는 방식으로 성과를 설계하는 사람이 되는 것입니다.

## 3. 인프라가 먼저다

이러한 것들이 모두 '개인 R&D'에 해당하는데, R&D를 꾸준히 하려면, 의지보다 조건이 먼저입니다.

**여유 시간**

R&D는 효율적으로 '사용하는' 시간이 아닙니다. 비효율적으로 '헤매는' 시간을 포함합니다. 설치하고, 설정하고, 막히고, 비교하고, 다시 시도하는 과정. 이게 확보되지 않으면 실험은 정보 소비로 끝납니다. 시간이 있어도 에너지가 없을 수 있습니다. 그렇다면 에너지를 먼저 채우세요.

**최소 생존 고정비**

생활비(OPEX)가 흔들리면, 도구 구독료나 실험비 같은 투자(CAPEX)는 곧바로 '불안한 지출'이 됩니다. 당장 내일 밥을 걱정하는데 AI 구독료를 내면 안 됩니다. 바닥이 안정돼야 꾸준히 투자하고, 투자한 만큼 학습이 누적됩니다.

**직접 써보는 시간**

남의 의견과 리뷰를 듣는 것도 중요하지만, 결국 차이는 직접 써보는 시간에서 납니다.

"그 앱 잘 나간다더라"를 아는 사람은 많습니다. 실제로 다운로드해 써보는 사람은 드뭅니다. 더 드문 사람은 그 경험을 관찰과 분석의 대상으로 삼습니다. 온보딩에서 사용자를 어느 지점에서 붙잡는지, 어떤 입력을 요구하고 어떤 순간에 결제를 유도하는지, 핵심 기능이 어떤 흐름으로 반복 사용을 만드는지. 이런 동작 패턴을 읽는 겁니다.

개인 R&D는 "유행을 아는 것"이 아닙니다. 유행을 직접 만져보고 구조를 분해해 이해하는 습관에서 출발합니다.

## 4. 모델의 "지그재그 지능"을 이해하라

### 최신 모델이 항상 모든 면에서 좋은 게 아닙니다

흔한 오해가 있습니다. "새 모델 = 모든 면에서 업그레이드"

현실은 다릅니다. Claude Sonnet 4는 코드 생성이 좋아졌지만 창작 글은 오히려 딱딱해졌습니다. GPT-4o는 속도가 빨라졌지만 복잡한 추론은 4-turbo가 나을 때가 있습니다. Opus 4.5는 깊은 분석에 좋지만 간단한 작업엔 과하고 느립니다.

모델마다 "잘하는 것"과 "못하는 것"이 다르고, 버전마다 "좋아진 것"과 "오히려 나빠진 것"이 있습니다.

> 이걸 모르면: 모든 작업에 최신 모델 → 돈 낭비, 속도 낭비. 모든 작업에 싼 모델 → 품질 낭비.
>
> 이걸 알면: 분류/추출 → Haiku (빠르고 쌈). 일반 작업 → Sonnet (균형). 복잡한 판단 → Opus (확실할 때만).
>
> "지그재그 지능"을 체감으로 아는 사람만 같은 비용으로 2-3배 효율을 냅니다.

### 더 무서운 이야기: 천만 원짜리 모델

지금 우리가 쓰는 건 "공개된 모델"입니다. Anthropic, OpenAI 내부에서는 더 좋은 모델을 쓰고 있습니다.

언젠가 이런 일이 생깁니다. "Opus 5 Pro: 월 천만 원, 기업 전용"

대부분의 반응은 "미쳤네, 누가 그걸 써"입니다. 하지만 누군가는 계산합니다. "천만 원 쓰면 1억 넘는 부가가치 만들 수 있는데?"

변호사가 천만 원 모델로 월 50건 서면을 자동화하면 절약 5천만원입니다. 투자사가 천만 원 모델로 딜소싱 분석을 하면 좋은 딜 하나 잡으면 수십억입니다. 컨설팅이 천만 원 모델로 리서치를 자동화하면 프로젝트 3개를 더 수주할 수 있습니다.

격차가 벌어지는 지점입니다. 월 2만원도 아까운 사람. 월 20만원 쓰면서 감 잡는 사람. 월 200만원 쓰면서 시스템 만드는 사람. 월 2000만원 써서 사업 굴리는 사람. 각 단계 사이에 10배씩 격차가 생깁니다.

지금 2만원 단계에서 "비싸다"고 멈추면, 나중에 200만원 단계가 열렸을 때 올라탈 수가 없습니다. 지금의 학습은 "지금 효율"만을 위한 게 아닙니다. "더 강력한 도구가 나왔을 때 쓸 수 있는 사람"이 되기 위한 것입니다.
