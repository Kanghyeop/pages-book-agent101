## 주장

- AI를 "쓰는 사람"이 아니라, AI와 함께 성과를 설계하는 사람이 되십시오.

## 사례: 변호사 A씨

```
동료 10명 중 Opus에 100만원+ 직접 써본 사람: 0명
반복 상담·서면 작성을 에이전트로 자동화한 사람: 0명
→ 그 교집합은 A씨 혼자
→ 대체 불가능한 위치
```

## 설명

### 인프라가 먼저입니다

의지로 되는 게 아닙니다. 조건이 갖춰져야 합니다.

#### 1. 여유시간

```
R&D는 효율적으로 '사용하는' 시간이 아닙니다.
비효율적으로 '헤매는' 시간을 포함합니다.

· 설치하다 막힘
· 설정하다 에러
· 비교하다 혼란
· 다시 시도하다 실패

이 과정이 확보되지 않으면
실험은 결국 "정보 소비"로 끝납니다.

"오늘 Claude 4.5 나왔대" → 트윗 보고 끝
vs
"오늘 Claude 4.5 나왔네" → 직접 써보고 비교

시간은 있어도 에너지가 없을 수 있습니다.
에너지를 먼저 채우십시오.
수면, 운동, 식사. 기본이 무너지면 R&D는 사치입니다.
```

#### 2. 고정비 안정

```
생활비(OPEX)가 흔들리면
도구 구독료, 실험비, 학습비 같은 CAPEX는
곧바로 "불안한 지출"이 됩니다.

당장 내일 밥을 걱정하는데
월 2만원 AI 구독료가 아깝게 느껴집니다.
100만원 API 실험비는 상상도 못합니다.

순서:
1. 최소 생존 고정비 확보 (월세, 식비, 보험)
2. 그다음 학습 예산 책정
3. 그래야 꾸준히 투자
4. 투자한 만큼 학습 누적

바닥이 안정돼야 쌓을 수 있습니다.
```

#### 3. 직접 써보는 시간

```
남의 의견과 리뷰를 듣는 것도 중요하지만
결국 차이는 "직접 써보는 시간"에서 납니다.

대부분의 사람:
"칼로리 앱 나왔다더라, 잘 나간다더라"
→ 알고 넘어감

드문 사람:
→ 실제로 다운로드해서 써봄

더 드문 사람:
→ 그 경험을 관찰과 분석의 대상으로 삼음

예를 들어:
· 온보딩에서 사용자를 어디서 붙잡는지
· 어떤 입력을 요구하고 언제 결제를 유도하는지
· 핵심 기능이 어떤 흐름으로 반복 사용을 만드는지

한 단계 더:
· 약관에서 데이터 수집 항목 확인
· 어떤 기능이 모델 기반인지 추정
· 외부 API 붙였는지 구조 파악

개인 R&D는 "유행을 아는 것"이 아니라
"유행을 직접 만져보고 구조를 분해하는 습관"입니다.
```

## 할 일 6가지

### □ 퍼스널 브랜딩

```
직장인이라도 조직 안의 역할에만 자신을 가두지 마십시오.

"나는 마케팅팀 대리입니다" ← 이건 직함
"저는 B2B SaaS의 콘텐츠 마케팅을 하고,
 특히 기술 제품을 비전공자에게 설명하는 걸 잘합니다" ← 이게 브랜딩

왜 필요한가:
· 이직할 때 설명이 쉬워집니다
· 협업 요청이 들어옵니다
· 변화가 잦은 시대에 내 가치를 안정적으로 전달

포장이 아닙니다. 생존 기술입니다.
```

### □ AI 유료 구독

```
무료 체험의 한계:
· "오 신기하네" → 끝
· 사용량 제한에 걸림
· 진짜 업무에 못 써봄
· 가능성만 보고 종료

유료 구독의 효과:
· "돈 냈으니 써야지" → 반복 사용
· 반복하니까 요령 생김
· 요령 생기니까 실력 됨
· 실력 되니까 성과 남

월 2만원이 아깝다고요?
그 2만원으로 월 10시간 절약하면 시급 2천원
→ 최저시급보다 싸게 일 시키는 겁니다
```

### □ 월 10~100만원 학습예산 집행

```
중요: 지출 자체가 목적이 아닙니다.

나쁜 예:
· 강의 질렀는데 안 봄
· API 크레딧 샀는데 안 씀
· 책 샀는데 안 읽음
→ 지출만 있고 학습 없음

좋은 예:
· 10만원 API로 자동화 실험 → 뭐가 되고 안 되는지 앎
· 50만원 들여 콘텐츠 제작 → 조회수/반응 데이터 얻음
· 100만원 Opus 써봄 → 모델 간 차이 체감

핵심 루프:
비용 → 실험 → 교훈 → 성과 → 재투자

"부가가치를 만드는 사람"이어야 합니다.
돈을 쓰는 사람이 아니라, 돈을 굴리는 사람.
```

### □ 신모델 나오면 직접 비교

```
"GPT-5 나왔대" → "아 그래?" → 끝
이러면 영원히 감 못 잡습니다.

해야 할 것:
1. 나오자마자 직접 써봅니다
2. 같은 프롬프트로 이전 모델과 비교합니다
3. 뭐가 달라졌는지 기록합니다

예시:
"Claude Sonnet 4 vs Sonnet 3.5"
· 코드 생성: 4가 더 정확 (체감 20%)
· 긴 문서 요약: 비슷
· 가격: 4가 조금 비쌈
· 결론: 코드 작업은 4, 단순 작업은 3.5

이 체감이 쌓이면:
· 같은 비용으로 더 좋은 결과
· 작업별 최적 모델 선택 가능
· 기술 변화가 업무에 미치는 영향 예측 가능

모델 간 차이를 모르면
"그냥 제일 비싼 거 쓰자" → 돈 낭비
"그냥 제일 싼 거 쓰자" → 품질 낭비
```

### □ 코딩 에이전트 활용

```
Claude Code 같은 도구로
비개발자도 자동화를 만들 수 있는 시대입니다.

BUT 만드는 것만으론 부족합니다.

나쁜 예:
· 프롬프트로 한 번 처리 → 끝
· 다음에 비슷한 일 → 또 처음부터

좋은 예:
· 프롬프트로 처리 → 템플릿으로 저장
· 반복되는 패턴 → 체크리스트로 정리
· 자주 쓰는 흐름 → 자동화 스크립트로 변환

"한 번의 성과"를 "반복 가능한 역량"으로 전환하십시오.

예시:
1회성: "이 PDF 요약해줘" → 결과 받음 → 끝
재사용: "PDF 요약 프롬프트 + 출력 형식 + 검증 기준"
        → 다음에 바로 적용 가능
        → 다른 사람도 쓸 수 있음
```

### □ 작은 자동화 하나 운영

```
"만들기"와 "운영"은 완전히 다른 기술입니다.

만들기:
· 한 번 성공하면 끝
· 예쁜 데모 가능
· "와 됐다!" 순간의 쾌감

운영:
· 매일 돌아가야 함
· 장애 나면 고쳐야 함
· 예외 케이스 처리해야 함
· 비용 모니터링해야 함
· 품질 떨어지면 개선해야 함

예시 - 회의록 자동화:
만들기: 녹음 → STT → 요약 파이프라인 구축 (1주)
운영:
· 월요일: 녹음 파일 형식 바뀌어서 에러
· 화요일: 긴 회의라 요약 품질 떨어짐
· 수요일: 비용 확인하니 예상보다 많이 나감
· 목요일: 팀원이 "액션아이템이 빠졌어요"
· 금요일: 프롬프트 수정해서 재배포

이 "운영 경험"이 쌓일 때 진짜 실력이 됩니다.

시작 추천:
· 규모: 나 혼자 또는 2-3명 팀
· 범위: 주 1회 정도 돌아가는 단순한 것
· 기간: 최소 3개월 운영

"자동화 만들 줄 알아요" vs "자동화 3개월 운영해봤어요"
후자가 10배 가치 있습니다.
```

## 모델의 "지그재그 지능"을 이해하라

### 최신 모델이 항상 모든 면에서 좋은 게 아닙니다

```
흔한 오해:
"새 모델 = 모든 면에서 업그레이드"

현실:
· Claude Sonnet 4: 코드 생성 좋아짐, 창작 글은 오히려 딱딱해짐
· GPT-4o: 속도 빨라짐, 복잡한 추론은 4-turbo가 나을 때 있음
· Opus 4.5: 깊은 분석 좋음, 간단한 작업엔 과하고 느림

모델마다 "잘하는 것"과 "못하는 것"이 다르고
버전마다 "좋아진 것"과 "오히려 나빠진 것"이 있습니다.

이걸 모르면:
· 모든 작업에 최신 모델 → 돈 낭비, 속도 낭비
· 모든 작업에 싼 모델 → 품질 낭비

이걸 알면:
· 분류/추출 → Haiku (빠르고 쌈)
· 일반 작업 → Sonnet (균형)
· 복잡한 판단 → Opus (확실할 때만)
· 코드 → 이번 버전이 나은지 이전이 나은지 체크

"지그재그 지능"을 체감으로 아는 사람만
같은 비용으로 2-3배 효율을 냅니다.
```

### 더 무서운 이야기: 천만 원짜리 모델

```
지금 우리가 쓰는 건 "공개된 모델"입니다.
Anthropic, OpenAI 내부에서는 더 좋은 모델을 쓰고 있습니다.

언젠가 이런 일이 생깁니다:
"Opus 5 Pro: 월 천만 원, 기업 전용"

대부분의 반응:
"미쳤네, 누가 그걸 써"

하지만 누군가는 계산합니다:
"천만 원 쓰면 1억 넘는 부가가치 만들 수 있는데?"

예시:
· 변호사: 천만 원 모델로 월 50건 서면 자동화 → 절약 5천만원
· 투자사: 천만 원 모델로 딜소싱 분석 → 좋은 딜 하나 잡으면 수십억
· 컨설팅: 천만 원 모델로 리서치 자동화 → 프로젝트 3개 더 수주

격차가 벌어지는 지점:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
· 월 2만원도 아까운 사람
· 월 20만원 쓰면서 감 잡는 사람
· 월 200만원 쓰면서 시스템 만드는 사람
· 월 2000만원 써서 사업 굴리는 사람

각 단계 사이에 10배씩 격차가 생깁니다.

지금 2만원 단계에서 "비싸다"고 멈추면
나중에 200만원 단계가 열렸을 때 올라탈 수가 없습니다.

지금의 학습은 "지금 효율"만을 위한 게 아닙니다.
"더 강력한 도구가 나왔을 때 쓸 수 있는 사람"이 되기 위한 것입니다.
```
