---

## 1. SW Tier (Part 2)

**정의**: AI 기술의 진화 단계를 나타내는 계층 구조

```
SW → AI → LLM → Tool-use(RAG) → Agent → A2A → MAS (→ AGI)
```

**시각화**:
```
[SW] ─→ [AI] ─→ [LLM] ─→ [Tool-use] ─→ [Agent] ─→ [A2A] ─→ [MAS]
 │       │       │         │            │          │         │
규칙    학습    언어      도구사용     자율행동   에이전트간  다중협업
기반    기반    이해      (RAG등)      (목표추구) 통신       시스템
```

**핵심 문장**: "각 단계는 이전 단계를 포함하며, 더 많은 자율성과 복잡성을 가진다."

---

## 2. IT Product (Part 2)

**정의**: 아이디어를 실제 서비스로 만드는 배포 기술 생태계

```
개발지식 (HTML, Python, Node, Next) + 배포 (Github, Streamlit, Vercel) + SaaS 연결
```

**시각화**:
```
[코드] ──→ [GitHub] ──→ [Streamlit/Vercel] ──→ [사용자]
  │           │              │
로컬개발    버전관리      자동배포/호스팅
```

**핵심 문장**: "개발자 1명, 서버 비용 0원, 24시간 서비스가 가능한 시대."

---

## 3. Agent Spectrum (Part 3)

**정의**: 에이전트의 자율성 수준을 5단계로 구분

```
L0 (Passive) → L1 (Tool) → L2 (ReAct) → L3 (Persistent) → L4 (Autonomous)
```

**시각화**:
```
[L0]────[L1]────[L2]────[L3]────[L4]
 │       │       │       │       │
호출    도구    반복    상태    자율
응답    사용    추론    유지    작동
 ↓       ↓       ↓       ↓       ↓
ChatGPT Function ReAct  Memory  Daemon
기본    Calling 패턴   Agent   Agent
```

**핵심 문장**: "레벨이 올라갈수록 인간은 실행자→검토자→감독자로 역할이 바뀐다."

---

## 4. 조직도 5블록 (Part 3)

**정의**: 조직을 5개 기능 블록으로 나누어 AI 도입 전략 수립

```
Executive (경영) / SharedServices (인프라) / Execution (실무) / Discovery (연구) / Front (고객접점)
```

**시각화**:
```
        ┌─────────────┐
        │  Executive  │ ← 의사결정
        └──────┬──────┘
               │
┌──────────────┼──────────────┐
│        SharedServices       │ ← Legal, Finance, HR
└──────────────┬──────────────┘
               │
    ┌──────────┴──────────┐
    │     Execution ★     │ ← 핵심 가치 창출
    └──────────┬──────────┘
               │
    ┌──────────┴──────────┐
    │  Discovery │ Front  │ ← 연구 / 고객접점
    └─────────────────────┘
```

**핵심 문장**: "AI 도입은 Front부터 시작하기 쉽고, Execution은 HITL 구조로 신중하게."

---

## 5. 기술 스택 (Part 4)

**정의**: 에이전트 개발에 사용되는 핵심 기술 계층

```
LLM API → LangChain/LangGraph → RAG/Memory → MCP/A2A
```

**시각화**:
```
[LLM API] ─→ [LangChain] ─→ [RAG] ─→ [MCP]
    │            │           │        │
  모델호출     워크플로우    지식검색   도구연결
              구성         증강       표준화
```

**핵심 문장**: "개발자와 대화하기 위한 최소한의 기술 어휘."

---

## 6. MAS (Part 4)

**정의**: 여러 에이전트가 협업하는 8가지 패턴

```
ReAct / Orchestrator-Worker / Router / Evaluator-Optimizer / MoA / CodeAct / Parallelization / HITL
```

**시각화**:
```
[단독] ReAct ────────────────────────→ [협업]
         │
    Orchestrator-Worker (팀장-팀원)
    Router (라우터)
    Evaluator-Optimizer (검토 루프)
    MoA (전문가 자문)
    CodeAct (코드 실행)
    Parallelization (병렬 처리)
    HITL (인간 승인)
```

**핵심 문장**: "혼자서 생각→행동을 반복하다가, 복잡해지면 역할을 나누고 협업한다."
