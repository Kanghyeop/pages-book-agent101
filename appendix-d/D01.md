## 13 | HITL → HOTL 전환

### (출판사 사례)

아래 내용을 설명하기 위해 출판사 사례를 활용하는 것으로 녹여내기

### HITL, HOTL, HOOTL 정의

전통적 정의 (트렌드코리아 2026에서 설명한 것 수준)

### HITL → HOTL

인간도 에이전트 (이 책에서 바라보는 시스템 정의)

서비스도 에이전트 (Tool과 Connector로 연결하면 됨)

인간이 있는 지점에서 병목이 발생하기 때문에 HITL이 HOTL이 되어야 한다

어시스턴트 서비스만 쓰면 10배성장은 만들 수 있어도 100배 성장은 만들 수 없다.

### HOTL이 되기 위해서 HITL이 먼저

기존에 사람도 못하던 새로운 일 에이전트 시킬게 아니라 기존에 반복되던 일

직접 해봐야 병목·인간 개입 지점 파악 가능

승인 단위를 키우며 병목 없애고자 하는 집착이 필요

### 방법론

```
1. 포착 - 입출력 정의 (시스템을 함수처럼)
2. 워크플로우 — 손으로 먼저, A4 용지
3. HITL — 직접 개입
4. 병목 제거 — 반복적·저리스크·고빈도 순
5. 예외만 — 높은 리스크, 감정 처리, 창의적 판단만 남김
```

## 14 | M-T-A 역할 선택

### MTA 역할모델 정의

```markdown
| 역할 | 설명 |
|------|------|
| M (감독자) | 시스템 전체 조율, 예외 판단, 기준 설정 |
| T (전문가) | 도메인 깊이, 품질 기준 정의 |
| A (프런트라인) | AI 도구 활용, 실행 |
```

### 방법론

```
기존 직무 ≠ 미래 역할. 본인의 선택이 결정.
제너럴리스트 시대: 여러 도메인 연결·조율 능력이 가치.
최소 구성: 도메인 전문가 + 엔지니어 (Hippocratic AI: 의사 2명 + 엔지니어 1명)
뷰티업계 AI 문제해결사 사례, 당연히 경력자 있으면 좋지만 몸값 높으니 2~3년차로 설명함
언러닝: 익숙한 방식 버리는 게 새로 배우는 것보다 어려움.
```

## 51 | 토큰 경제

써라. 보인다. 아껴라.

```markdown
1. 써라 — 등록금, 비용 감각 없으면 설계 못함
2. 보인다 — 어디서 새는지, 뭐가 비싼지
3. 아껴라 — 모델 계층화, 컨텍스트 압축, 캐싱, 배치
```

토큰이란? 토큰, 컨텍스트, 등 직결 개념. 에이전트의 밥, 산출물, 인건비

### 6가지 당장 할 것

1. 퍼스널 브랜딩 — 직함이 아니라 역량을 언어화
