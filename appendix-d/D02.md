## 혼란스러움이 생기는 상황과 이유

### 프레임워크가 없을 때 생기는 흔한 오해

- "에이전트"라는 단어가 여러 맥락에서 다르게 쓰임
- 기술 발전 단계와 자율성 레벨이 혼동됨
- 하나의 제품이 여러 관점에서 다르게 해석됨
- ChatGPT가 에이전트인가 아닌가? (관점에 따라 다름)

### 다섯 가지 프레임워크가 복합적으로 얽히는 문제

하나의 제품/사례를 설명할 때:
- "이건 어느 기술 단계인가?" (8-tier)
- "자율성은 어느 레벨인가?" (L0~L4)
- "조직 어디를 대체하는가?" (5블록)
- "내부 구조는?" (MAS 패턴)
- "어떻게 구현했나?" (기술스택)

비전공자에게 이 다섯 가지 관점이 동시에 작동하면 혼란스러움

---

## 다섯 가지 프레임워크

### 1. 기술발전의 흐름 (8-tier)

```
Computer → SW → AI → LLM → Tool-use → Agent → MAS → AGI
```

- 의존성 흐름: 앞 단계를 이해해야 다음 단계 이해 가능
- 고민: 책에서 어디까지 설명할 것인가

### 2. 에이전트 자율성 스펙트럼

```
L0 Passive Tool → L1 Assistant → L2 Agentic → L3 Agent → L4 Autonomous
```

- 핵심: 사람이 얼마나 개입하는가
- HITL → HOTL 전환과 연결

### 3. 인간 조직 구성과 대체시도 (버티컬 SaaS)

```
Executive / SharedServices / Execution / Discovery / Front
```

- 5블록 모델
- 어디를 먼저 자동화? → Front부터
- Execution은 HITL 필수

### 4. 에이전트 조직의 구성

- MAS 내부에서도 5블록과 유사한 구조 가능
- Orchestrator = Executive
- Worker = Execution
- 고민: 이 대응 관계가 적절한가

### 5. 구현을 위한 기술스택

- 레이어: 모델/인프라 → 프레임워크 → 애플리케이션
- 도구들: LangChain, Zapier, Claude Code 등
- 고민: 어디까지 설명해야 Y씨에게 유용한가

- LLM 정의가 Ch.42에서야 등장
- 메모리/도구 개념이 3중 중복

- Ch.46 패턴 8가지 과부하

재구성 범위:
- 부분 수정 vs 전면 재구성
- 결정 보류 중

- 축1: SW (LLM이 아닌 최적화 알고리즘)

- "에이전트 이전 단계에서의 전환"으로 프레이밍?
- 위치를 바꿔야 하나?
