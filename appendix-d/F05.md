> 에이전트를 구현하기 위해 알아야 할 5가지 핵심 계층

기술용어가 쏟아집니다. 쫄지 맙시다. 목표는 에이전트를 뚝딱 만드는 게 아닙니다. 개념의 지도를 그려서 상황을 읽고, 원하는 방향을 표현할 수 있게 되는 겁니다. 무기를 장착하기 위한 근력 운동입니다.

---

## 에이전트를 만들고 싶다면, 5가지를 알아야 합니다

```
┌─────────────────────────────────────────────┐
│  5. 손발: 외부 세계와 어떻게 연결하나?        │
├─────────────────────────────────────────────┤
│  4. 기억: 정보를 어떻게 저장하고 찾나?        │
├─────────────────────────────────────────────┤
│  3. 흐름: 복잡한 작업을 어떻게 구성하나?      │
├─────────────────────────────────────────────┤
│  2. 대화: 시스템끼리 어떻게 연결하나?        │
├─────────────────────────────────────────────┤
│  1. 두뇌: 어떤 모델을 쓸까?                  │
└─────────────────────────────────────────────┘
```

아래에서 위로 쌓아 올립니다. 두뇌가 있어야 대화하고, 대화해야 흐름을 만들죠.

---

## 1. 두뇌: 어떤 모델을 쓸까?

**왜 필요한가?**
에이전트의 판단력이 곧 LLM입니다. 모든 결정은 여기서 나옵니다.

**핵심 개념**
- LLM은 "다음에 올 단어"를 예측하는 기계입니다
- 수십억 권의 책을 읽은 사람에게 "이 문장 뒤에 뭐가 올 것 같아?"라고 묻는 셈이죠

**선택 기준**

| 요구사항 | 선택 |
|---------|------|
| 빠른 응답 필요 | 작은 모델 (Haiku) |
| 복잡한 추론 필요 | 큰 모델 (Opus) |
| 비용 절감 필요 | 작은 모델 + 라우팅 |

트레이드오프가 있습니다: 성능 ↑ = 속도 ↓ = 비용 ↑

---

## 2. 대화: 시스템끼리 어떻게 연결하나?

**왜 필요한가?**
에이전트는 혼자 일하지 않습니다. LLM 서버, 데이터베이스, 외부 서비스와 대화해야 합니다.

**핵심 개념**
- **API**: 프로그램이 다른 프로그램과 대화하는 방법
- **JSON**: 데이터를 주고받는 공용어

```json
{
  "name": "홍길동",
  "age": 30,
  "skills": ["기획", "분석"]
}
```

**알면 편한 것**
- SDK를 쓰면 API를 쉽게 호출할 수 있습니다
- 예: `openai.chat(...)` 한 줄로 GPT 호출

---

## 3. 흐름: 복잡한 작업을 어떻게 구성하나?

**왜 필요한가?**
단순한 질답을 넘어 여러 단계 작업이 필요합니다. "문서 읽기 → 요약 → 번역 → 저장" 같은 흐름이죠.

**핵심 도구**

| 도구 | 용도 | 비유 |
|------|------|------|
| LangChain | 순서대로 실행 | 레시피 |
| LangGraph | 분기/반복 | 게임 공략 |

**선택 기준**
```
작업이 직선적이면 → LangChain
"만약 ~이면"이 많으면 → LangGraph
```

LangChain은 A→B→C 순서로 진행합니다.
LangGraph는 조건에 따라 A→B 또는 A→C로 갈 수 있죠.

---

## 4. 기억: 정보를 어떻게 저장하고 찾나?

**왜 필요한가?**
LLM은 학습할 때 본 정보만 압니다. 우리 회사 문서, 최신 뉴스는 모릅니다.

**핵심 개념**
- **RAG** (검색 + 생성): 질문이 들어오면 관련 문서를 먼저 찾고, 그걸 참고해서 답합니다
- **벡터DB**: "이것과 비슷한 것"을 찾아주는 특수 데이터베이스

```
질문 → 관련 문서 검색 → LLM에게 문서와 함께 전달 → 정확한 답변
```

**선택 기준**

| 질문 유형 | 적합한 방식 |
|----------|------------|
| "X에 대해 알려줘" | RAG (유사도 검색) |
| "A와 관련된 B를 찾아줘" | GraphRAG (관계 탐색) |

---

## 5. 손발: 외부 세계와 어떻게 연결하나?

**왜 필요한가?**
텍스트만 생성하면 실제로 아무것도 할 수 없습니다. 파일을 저장하고, 이메일을 보내고, 데이터를 조회하려면 도구가 필요합니다.

**핵심 도구**

| 도구 | 역할 | 비유 |
|------|------|------|
| MCP | 도구 연결 표준 | 표준화된 매뉴얼 |
| A2A | 에이전트 간 협업 | 팀원과 대화 |

MCP가 있으면 에이전트가 도구 사용법을 자동으로 파악합니다.
A2A가 있으면 항공권 에이전트, 호텔 에이전트가 서로 협업하죠.

**보안 5원칙**

도구를 줄 때 반드시 고려해야 합니다:

| 원칙 | 한 줄 설명 |
|------|----------|
| 최소 권한 | 필요한 권한만 부여 |
| 샌드박싱 | 격리된 환경에서 실행 |
| 모니터링 | 모든 행동 기록 |
| 비용 제한 | 예산 상한선 설정 |
| 인간 승인 | 중요 작업은 확인 후 실행 |

## 활용 사례 (참고)

위 5계층으로 만든 에이전트 제품들이에요. 구현 기술이 아니라 활용 사례입니다.

| 제품 | 대상 | 특징 |
|------|------|------|
| Claude Code | 개발자 | 터미널 기반, 코딩 특화 |

두 제품 모두 로컬 폴더 접근 권한을 받아 자율적으로 작업합니다.

## 정리

에이전트 구현에 필요한 5가지 계층:

| 계층 | 핵심 질문 | 알아야 할 것 |
|------|----------|-------------|
| 1. 두뇌 | 어떤 모델? | LLM, 트레이드오프 |
| 2. 대화 | 어떻게 연결? | API, JSON, SDK |
| 3. 흐름 | 어떻게 구성? | LangChain, LangGraph |
| 4. 기억 | 어떻게 검색? | RAG, 벡터DB |
| 5. 손발 | 어떻게 행동? | MCP, A2A, 보안 |

개발자와 대화할 때 이 용어들을 알면 소통할 수 있습니다.
