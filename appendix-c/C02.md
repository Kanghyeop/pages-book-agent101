> 약 1,500 토큰으로 60,000+ 토큰의 전체 내용을 압축합니다.

---

## 핵심 논제

**"AI는 도구가 아니라 구조다."**

- 10배 생산성 = 도구 (ChatGPT, Copilot)
- 100배 생산성 = 구조 (에이전트 시스템, 자동화 워크플로우)
- HITL(Human-in-the-Loop) → HOTL(Human-on-the-Loop) 전환이 핵심
- AI는 좋은 회사를 더 좋게 만들지만, 나쁜 회사를 좋게 바꾸진 않는다

---

## 핵심 개념 관계망

```
[HITL] ─전환→ [HOTL]
   │              │
   ▼              ▼
매 단계 승인    루프 밖 감독
1:1 증강       구조적 자동화

[에이전트 = Brain + Senses + Tools + Memory + Goals]
     │
     ▼
[5-Block Model: Executive / SharedServices / Execution / Discovery / Front]
     │
     ▼
[M-T-A 역할: Manager(감독) / Technician(구축) / Augmented(실행)]
```

### 핵심 수식
- **에이전트 = 자율성 + 목표 지향 + 환경 상호작용**
- **SuperStaffing = 대체 아님, 처리 용량 무한 확장**
- **가치 = Execution Layer에서 발생 (Discovery 아님)**

## Part별 핵심

### Part 1: 필요한 이유
- 고용은 식는데 실적은 오른다 (에버노트, 클라나 사례)
- M-T-A 모델: 조직 내 역할 재정의
- Unlearning(기존 지식 버리기) + Upskilling(새 역량 습득)

### Part 2: 배경지식
- 모델 vs 서비스 vs Task 구분
- 6가지 서비스 카테고리 (Chat, Create, Search, Translate, Summarize, Analyze)
- 서빙/라우팅/프레임워크/플랫폼 구분

### Part 3: 에이전트 정의
- 학문적 정의 (Russell&Norvig, BDI, ReAct)
- 실용적 정의: Input→Processing→Output 시스템
- 5-Block 패턴 (병원, 로펌 등 도메인 대입)

### Part 4: 기술 스택
- LangChain(순차) vs LangGraph(분기/상태)
- Embedding→VectorDB→RAG→GraphRAG 계층
- MCP/A2A 프로토콜, ComputerUse/BrowserUse
- 멀티에이전트 패턴: ReAct, Orchestrator-Worker, Router, MoA

### Part 5: 실행
- 토큰 경제학: 손익분기 = 인건비 절감 > API 비용
- 개인 6가지 액션: Claude 구독, 영어 프롬프트, GitHub 세팅...
- 조직 라이트하우스: 3개월 Quick-win 프로젝트
- 4가지 전제: 데이터 접근, 의사결정 권한, 기술 지원, 실패 허용

### 부록 A: 실무 가이드
- Claude Code 사용법 (Plan mode, Auto-accept)
- 보안 체크리스트 (API Key, MCP, 비용폭탄, 권한관리)
- 온프레미스 vs 클라우드 판단 기준

### 부록 B: 심화 사례
- Hippocratic AI: 의료 특화, $9/시간, Constellation 아키텍처
- Wonderful AI: 문화 특화, Sub-500ms, Local by Design
- Claude Code 개발자 꿀팁: 터미널 권한 = 핵심 차별점

## 핵심 용어 사전

| 용어 | 정의 |
|------|------|
| HITL | Human-in-the-Loop. 매 단계 사람 승인 |
| HOTL | Human-out-of-the-Loop. 사람이 루프 밖에서 감독 |
| SuperStaffing | 대체 아닌 처리 용량 무한 확장 |
| M-T-A | Manager-Technician-Augmented 역할 모델 |
| 5-Block | Executive/SharedServices/Execution/Discovery/Front |
| LangChain | 순차적 체인 기반 에이전트 프레임워크 |
| LangGraph | 상태 기반 그래프 에이전트 프레임워크 |
| MCP | Model Context Protocol. 도구 연결 표준 |
| A2A | Agent-to-Agent. 에이전트 간 통신 |
| RAG | Retrieval-Augmented Generation |
| 라이트하우스 | 3개월 Quick-win 파일럿 프로젝트 |
| Constellation | 다중 모델 상호검증 아키텍처 (Hippocratic AI) |

## 책의 차별화 포인트

1. **불안→행동 전환**: 신기술 책의 불안 증폭을 액션플랜으로 돌파
2. **시각자료 중심**: 다이어그램, 표, mermaid로 직관적 이해
3. **사례 패턴화**: 100+ 사례를 5-Block 패턴으로 재구성
4. **직설적 톤**: "될 것/안 될 것" 명확히 구분
