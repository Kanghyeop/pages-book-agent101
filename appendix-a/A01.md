## 배경지식으로 알아야 하는 것

### 폴더와 터미널을 통한 컴퓨터 제어 개념

- Claude Code는 “코드 편집기 앱(VisualStudio 또는 Cursor)”이라기보다 터미널에서 실행되는 도구라서, 먼저 폴더(디렉터리) 구조와 “현재 내가 작업 중인 위치” 개념이 필요합니다. 여기서 말하는 프로젝트 루트는 번역기 앱 같은 작업 폴더의 최상단을 뜻하고, Claude Code는 터미널이 가리키는 그 폴더를 기준으로 파일을 읽고 만들고 수정합니다. 그래서 기본적으로 `cd`로 프로젝트 폴더로 이동한 뒤, 그 자리에서 실행하는 흐름을 이해해야 합니다.

### Windows(Powershell) / macOS(Terminal) 환경설정 및 Claude Code 설치 + 터미널에서 claude code에 접속하는법

- 운영체제별로는, Windows에서는 보통 PowerShell(또는 Windows Terminal)에서 명령을 실행하고, macOS에서는 Terminal(기본 셸은 zsh)에서 동일한 방식으로 명령을 실행합니다. “환경설정 및 설치”는 결국 터미널에서 설치 명령을 실행하고, 설치가 끝난 뒤 `claude` 같은 커맨드가 인식되는지 확인하는 과정입니다. 설치가 완료되면, 프로젝트 폴더에서 `claude`를 실행해 Claude Code 세션을 시작하게 됩니다.

### GitHub 계정 생성 및 GitHub 최소 기본개념

- 마지막으로 GitHub는 Claude Code 자체 필수는 아니지만, 강의/프로젝트 흐름에서 거의 항상 쓰이기 때문에 **최소 개념**은 알아두는 게 좋습니다. GitHub 계정은 원격 저장소를 만들기 위한 준비이고, 로컬에서 작업한 변경사항을 **commit**으로 “기록”한 다음, **push**로 “GitHub에 업로드”하는 정도만 알면 일단 진행이 됩니다. 즉, “터미널에서 프로젝트 폴더로 이동 → `claude`로 작업 → 변경사항 commit/push”가 기본 루프입니다.
- **→ 좀 감 잡혔다 싶어서 본격적으로 해야겠다 싶으면 깃헙 레포를 설정하고 늘 commit/push를 하시는 것이 좋습니다. 그러면 변경사항 이력이 히스토리로 저장이 되기 때문에 언제든지 롤백(되돌아가기)이 가능합니다.**

## 알면 좋은 것 (필수)

- 요금제
    - Claude Code는 구독 요금제에 따라 사용 가능한 범위(모델/기능)와 체감 사용량 한도가 달라집니다. 그래서 “내가 어떤 요금제인지”를 알고 시작하는 게 필요합니다.
- 모델 → **처음부터 Opus만 사용하고, 토큰 부족할때 불필요한 작업에만 낮추기**
    - Opus: 가장 강력한 성능 (동작할때 토큰 사용량 많음)
    - Sonnet: 성능과 속도의 균형 (무난하나 코드짤때는 적당한 성능발휘)
    - Haiku: 가장 빠르고 가벼움 (동작할때 토큰 사용량 적음)
- 토큰/컨텍스트/사용량 제한/최적화
    - 토큰:
        - Claude가 텍스트(질문/답변/코드/파일)를 처리할 때 쓰는 “글자 단위의 비용” 같은 것(길수록 더 많이 듦)
    - 컨텍스트:
        - Claude가 한 번에 참고하는 “기억 범위” (지금까지 대화 + 첨부/읽은 파일 포함)
        - 너무 길어지면 알아서 중간중간 압축을 한다.
    - 사용량제한:
        - 이 페이지는 띄워놓고 의식하자. https://claude.ai/settings/usage
        - 다섯시간마다 쓸수있는 ‘세션 사용량 한도’가 리필된다.
        - 일주일마다 쓸수있는 ‘주간 사용량 한도’가 리필된다.
    - 최적화:
        - 굳이 LLM을 쓰지 않아도 되는 작업은 Script를 쓴다.
        - 단순한 코드를 대규모로 작성시에는 Opus 말고 Sonnet을 쓴다.
        - 그 외 Production을 하기 위해서는 요구되는 최적화 개념들이 있으나 Entry 레벨에서는 위의 두개 정도의 개념들만 일단 알아도 충분함.
- 모드(Default mode)
    - 말 그래도 기본 모드 설정이다.
        - **Shift Tab을 눌러서 모드를 전환한다.**
        - **왠만한 모든 작업을 할때 먼저 플랜모드를 거치시길 추천합니다. 대부분! 모두!**
    - 두가지가 있다.
        - Action: 계획 설명을 길게 하지 않고 바로 파일 수정·명령 실행 같은 실제 작업을 진행하는 모드(속도 중심, 대신 실수 리스크↑)
        - Plan: 먼저 “무엇을/왜/어떻게 바꿀지”를 계획으로 정리하고, 실행·수정은 그 다음에 진행하는 모드(안전/설계 중심
- 권한(Rules)
    - 클로드코드가 알아서 내 컴퓨터 포맷해버리면 안되기 때문에 권한을 제어한다 : Rules는 “매번 관리하는 기능”이 아니라, Auto-Accept를 쓰는 사람일수록 ‘딱 한 번의 대형 사고’를 막기 위해 최소한으로 걸어두는 안전벨트다.
    - 세가지가 있다.
        - allow (무조건 허용)
        - ask (무조건 물어봄)
        - deny (무조건 차단)

## CLI 주요 옵션/플래그

### 기본

```bash
claude
```

### 저는 이렇게 씁니다

```bash
claude --model opus --permission-mode bypassPermissions
claude --model opusplan --permission-mode bypassPermissions
```

### 기존 세션 대화 이어가기

```bash
claude --continue
```

### 모델

```bash
claude --model opus #실행+계획 모두 Opus <- 추천
claude --model opusplan #계획은 Opus, 실행은 Sonnet <- 토큰 부족할 때
claude --model sonnet #실행+계획 모두 Sonnet <- 토큰 없을 때
```

### 권한

그냥 기본으로는 default 모드로 켜고 위험한 작업 안할때는 bypassPermission 해도 됩니다. **초보자 기준으로는 bypassPermission 쓰지마세요. 본인이 책임을 질 수 있을 지식이 쌓이면 쓰세요.**

```bash
claude --permission-mode default = claude #기본 권한 모드(처음 쓰는 도구는 승인 질문함). claude만 실행한 것과 동일
claude --permission-mode acceptEdits #파일 편집(Edit/Write) 권한을 자동 승인하고 진행
claude --permission-mode plan #계획/분석만 수행(명령 실행·파일 수정 불가)
claude --permission-mode dontAsk #미리 허용(allow)된 것만 실행, 그 외는 질문 없이 자동 거부
claude --permission-mode bypassPermissions #권한 프롬프트를 전부 건너뛰고 실행(위험, 격리 환경에서만 권장)
```

### 환경설정 파일 → 안써도 됨

`settings.json` = Claude Code의 “기본 동작(권한/도구/환경)”을 저장하는 설정 파일

- 팀 공유: .claude/settings.json
- 개인 로컬: .claude/settings.local.json

`.env` 환경변수 = 실행 시 주입되는 “비밀값/환경값(API 키 등)”

`.gitignore` = 커밋되면 안 되는 파일 목록(특히 비밀값)

### CLAUDE.md = 메모리/컨텍스트/규칙 → 안써도 됨

settings.json과의 차이

- `CLAUDE.md`: “무엇을/어떻게” 일하게 할지(지침/규칙/맥락)
- `settings.json`: 권한/환경변수/도구 동작 같은 “실행 환경 설정”

“쓰는 게 확실히 이득”인 경우

- 팀 프로젝트(온보딩/일관성 필요)
- 레포가 크거나 모노레포
- 반복되는 작업/규칙(테스트, 포맷, 커밋 규칙, 배포 절차)이 많음

프로젝트 시작할때 /init으로 시작해도 되고, 그렇게 하면 이런거 자동으로 생김

### 소리나게 알람설정 → 안써도 됨

쓰다보면 ‘클로드코드 작업 끝나거나 요청이 필요할때 내게 알려주면 좋겠는데’ 싶은 순간이 옴.

```
  설정 내용:

  "hooks": {
    "Notification": [
      {
        "matcher": "",
        "hooks": [{
          "type": "command",
          "command": "powershell -Command \"(New-Object System.Media.SoundPlayer 'C:\\Windows\\Media\\Alarm02.wav').PlaySync()\""
        }]
      }
    ],
    "Stop": [
      {
        "matcher": "",
        "hooks": [{
          "type": "command",

        }]
      }
    ]
  }

  동작 방식:
  - Notification 이벤트: Alarm02.wav 재생
  - Stop 이벤트 (채팅 끝날 때): Alarm01.wav 재생

  둘 다 Windows 기본 사운드 파일(C:\Windows\Media\)을 PowerShell로 재생하는 방식입니다.
```
